# 里程碑 2 (M2) 详细开发计划：创作推演引擎与核心 AI 对话流

## 阶段目标
为系统注入“灵魂”。利用双轨 Prompt 设计，在 Zep 体系下构建基于动态记忆的 AI 导演引擎 (Director AI)。

## 预估周期
4 - 5 周

## 第一周：多用户动态 Session 与时间线设计
* **目标：** 隔离不同用户的状态，并奠定随时可以进行“剧情改写 / 节点回溯”的基础。
* **具体任务：**
  * [x] **Session 记录系统开发**：每个玩家对应一个新建的 Zep Session，记录所有的游玩流水。
  * [x] **“剧情书签 (Bookmark)”机制**：在数据库层记录每一次交互操作对应的 Zep Session 状态。
  * [x] **读档/分支创建接口**：当玩家选择回溯到某个具体章节/对话节点时，利用 Zep 接口复制此节点之前的所有 Context 到新的 Session 中，形成“平行宇宙”。

## 第二周：双轨 Director Prompt 体系搭建
* **目标：** 基于不同的玩法模式（沙盒 vs. 收束）设计针对性的提示词工程。
* **具体任务：**
  * [x] **核心组件封装**：组装上下文，使得 Director AI 在每次请求时都能拿到 -> `System Prompt + Zep长摘要 + Zep人物关系 + 近期5-10轮对话`。
  * [x] **沙盒模式 (A 模式) Prompt 编写测试**：重点测试 AI 是不是会过度替玩家决定、是否能客观反应世界法则（如玩家的逆天行为是否会遭到世界抵制）。
  * [x] **收束模式 (B 模式) Prompt 编写与 Waypoints 解析**：向 Prompt 植入“接下来的剧情路标 (Waypoints)”，观察大模型能否平滑地引导走向而不是生硬地跳跃。

## 第三周：基于 LLM 的同步解构引擎与意图路由
* **目标：** 实现玩家自然语言输入的“即时解析+剧情推演”一体化，为逻辑层提供结构化抓手。
* **具体任务：**
  * [x] **导演 AI 结构化输出协议 (Joint Output Protocol)**：设计并测试 Director LLM 的 JSON 返回规范。确保单次请求同时返回：`story_text` (剧情)、`user_intent_summary` (玩家意图分解)、`world_impact` (世界状态影响) 及 `ui_hints` (界面渲染提示)。
  * [ ] **指令注入防御层 (Safety Guard)**：利用 LLM 意图解析，自动识别并拦截玩家试图“越狱”修改系统设定或跳出剧情框架的非法指令。
  * [x] **多层级解析逻辑**：若玩家输入包含物理动作（如“挥剑”），系统需同步输出 `combat_event` 标识，以便后续里程碑对接数值系统或图谱变更。

## 第四周：基于解构数据的沉浸式 UI 渲染
* **目标：** 利用后端返回的 JSON 结构化数据，实现超越传统聊天的氛围感交互。
* **具体任务：**
  * [x] **多态文本渲染组件**：根据 `user_intent_summary` 和 `story_text` 中的标签，对“旁白、对话、心理、系统提示”进行差异化排版（字体、颜色、背景动效）。
  * [x] **动态状态实时反馈**：若后端返回 `world_impact` 包含好感度或环境变化，前端即时弹出微型提示框或改变当前场景氛围色。
  * [x] **交互式动作反馈**：为玩家的“动作”和“对话”提供不同的输入确认感，增强第一人称扮演的反馈回路。

## 第五周：基于解构特征的叙事节奏控制器 (Pacing Controller)
* **目标：** 通过分析玩家意图的“有效性”和“广度”，智能干预剧情进度。
* **具体任务：**
  * [x] **意图饱和度算法**：后台分析 `user_intent_summary`。若玩家连续多轮只有 `social_chat` 而无实质性 `action` 推动，判定为剧情停滞。
  * [x] **动态危机注入系统**：根据停滞判定，在下一轮 Director Prompt 中强制混入 `pacing_event`。AI 导演在生成 JSON 时，必须包含一个由于当前局势导致的“突发冲突”或“环境变化”。
  * [ ] **图谱变更自动触发器**：根据解构出的 `world_impact` 字段，判定是否需要对 Zep 初始图谱进行原子化的增删改，实现“行为即改变”。

## 第六周：终端集成测试与调优
* **目标：** 不依赖复杂 UI，深度测试大模型的响应稳定性。
* **具体任务：**
  * [ ] **CLI / Postman 手动体验**：团队人员通过终端直接模拟一个特定的人物开局。
  * [ ] **幻觉与遗忘修复**：观察长对话中 AI 是否会出现“OOC”、“忘记装备”的情况，如有，调整 Zep 检索窗口和摘要提取密度。

## 当前进度摘要（滚动更新）
* **已完成**
  * 会话/书签/分支全链路可用，支持从章节与书签开启平行宇宙。
  * Director 双轨 Prompt + JOP 返回协议落地，支持 `story_text/user_intent_summary/world_impact/ui_hints`。
  * `combat_event` 标识与 UI 提示已接入。
  * 起始章节注入（`start_chapter_id/title/preview`）已接入 Director system prompt。
  * UI 多态渲染、世界变化提示、输入模式按钮已完成。
  * 叙事节奏控制器（停滞检测 + 危机注入）已完成。
  * 图谱 `source/priority` 预留策略与实体类型归一化已完成。
  * 原始剧情时间线交互优化（可折叠、横向滚动、自动刷新）。
* **待补**
  * Safety Guard（指令注入防御层）尚未实现。
  * 回归脚本（50 回合 + 书签再分支 + 模式切换）未补齐。
  * 图谱变更自动触发器（`world_impact` → 图谱原子化更新）未完成。

## 验收标准
1. 能够通过类似终端或 API 工具连续交互 50 个回合以上不发生明显的前后逻辑断裂和遗忘。
2. 双模式体现差异：相同动作在 A 模式下导致无序发展，在 B 模式下最终被剧情平滑“拉回主线”。

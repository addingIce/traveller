# 里程碑 1 (M1) 详细开发计划：数据基建与知识萃取核心

## 阶段目标
让系统具备“读懂”小说并提取长程结构化信息的能力，为后续的剧情推演提供基础的数据和图谱支持。

## 预估周期
3 - 4 周

## 第一周：系统初始化与环境准备
* **目标：** 完成开发框架的选择、后端工程搭建和基础设施部署。
* **具体任务：**
  * [ ] **选型与建项**：采用 Python (FastAPI/Django) 作为后端主框架。
  * [ ] **Zep 环境部署**：通过 Docker 或源码在开发环境拉起 Zep 服务，验证连接通道。
  * [ ] **大模型接口对接**：集成主流大模型（如 OpenAI、vLLM 或国产模型）API，封装基础 `generate` 和 `chat` 函数。
  * [ ] **数据库设计**：初步建立关系型数据库（如 PostgreSQL/MySQL），用于存储基本的小说元数据和简单的用户鉴权。

## 第二周：文本处理与摄入 Pipeline
* **目标：** 实现从纯文本格式到 Zep 可识别的文档集合的转换与摄入。
* **具体任务：**
  * [ ] **文本清洗工具开发**：编写清洗脚本，支持主流小说格式（TXT/EPUB），处理换行、特殊符号和长句截断。
  * [ ] **文档切块 (Chunking) 策略**：按章节和自然段设置合理分块逻辑（不破坏语义连贯性）。
  * [ ] **文本摄入**：通过 Zep API 创建 Document Collection，将分块的小说文本与 Metadata (如小说名、卷号、章节号) 批量写入，确保向量化检索功能可用。

## 第三周：图谱提取与人工干预
* **目标：** 完成小说世界观内的核心人物、地点、派系与关键物品关系的提取。
* **具体任务：**
  * [ ] **Graph Extraction 功能激活与测试**：调用 Zep 的 Knowledge Graph Extraction 功能。
  * [ ] **定制化 Prompt (可选)**：针对网文特定语境（如：宗门、功法、法宝），定制提取规则，提升图谱提取质量。
  * [ ] **数据清洗和补全脚本**：针对 AI 可能会错漏的少部分核心人物或错误的关系线，提供简单的增删改逻辑，保证“原著”的基础设定完全准确。

## 第四周：前端 MVP (档案库展示)
* **目标：** 实现简单的 Web 前端，将底层抽象的数据内容具象化展现给开发者与初期体验者。
* **具体任务：**
  * [ ] **Vue/React 基础页面搭建**：完成基本页面路由。
  * [ ] **数据可视化集成**：引入 Echarts.js 或 AntV G6 渲染关系图表。
  * [ ] **检索能力 UI 化**：添加一个搜索框，使得搜索“张无忌”时，能够基于后端接口渲染出他在当前小说库中相关联的人物节点和介绍文本。

## 验收标准
1. 能成功摄入一本百万字小说并支持大段文本语义搜索。
2. 能在一个页面内通过输入角色名字直观展示其人物关系和设定。

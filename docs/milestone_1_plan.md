# 里程碑 1 (M1) 详细开发计划：数据基建与知识萃取核心

## 阶段目标
让系统具备“读懂”小说并提取长程结构化信息的能力，为后续的剧情推演提供基础的数据和图谱支持。

## 预估周期
3 - 4 周

## 第一周：系统初始化与环境准备
* **目标：** 完成开发框架的选择、后端工程搭建和基础设施部署。
* **具体任务：**
  * [x] **选型与建项**：采用 Python (FastAPI/Django) 作为后端主框架。
  * [x] **Zep 环境部署**：通过 Docker 或源码在开发环境拉起 Zep 服务，验证连接通道。
  * [x] **大模型接口对接**：集成主流大模型（如 OpenAI、vLLM 或国产模型）API，封装基础 `generate` 和 `chat` 函数。
  * [x] **数据库设计**：初步建立关系型数据库（如 PostgreSQL/MySQL/Neo4j），用于存储元数据和图谱。

## 第二周：文本处理与摄入 Pipeline
* **目标：** 实现从纯文本格式到 Zep 可识别的文档集合的转换与摄入。
* **具体任务：**
  * [x] **文本清洗工具开发**：编写清洗脚本，支持主流小说格式（TXT），处理换行和分块。
  * [x] **文档切块 (Chunking) 策略**：按自然段设置合理分块逻辑。
  * [x] **文本摄入**：通过 Zep API 将切块内容与 Session 关联，确保持久化与检索可用。

## 第三周：图谱提取与人工干预
* **目标：** 完成小说世界观内的核心人物、地点、派系与关键物品关系的提取。
* **具体任务：**
  * [x] **Graph Extraction 功能激活与测试**：集成 Graphiti 核心抽取链路。
  * [x] **定制化 Prompt (可选)**：针对中文小说语境，定制语言一致性提取规则（中文化）。
  * [x] **数据清洗和补全脚本**：支持直接通过后端 API 绕过摘要获取物理图谱，保障展示质量。

## 第四周：前端 MVP (档案库展示)
* **目标：** 实现简单的 Web 前端，将底层抽象的数据内容具象化展现给开发者与初期体验者。
* **具体任务：**
  * [x] **React 基础页面搭建**：完成基本页面布局与交互。
  * [x] **数据可视化集成**：引入 AntV G6 渲染关系图表。
  * [ ] **检索能力 UI 化**：完善侧边栏与搜索功能，支持通过角色名字定位节点。

## 验收标准
1. 能成功摄入一本百万字小说并支持大段文本语义搜索。
2. 能在一个页面内通过输入角色名字直观展示其人物关系和设定。

# 里程碑 3 (M3) 详细开发计划：互动游玩端与沉浸式用户体验

## 阶段目标
为终端用户提供一个拥有丰富包裹和代入感的操作交互层。把引擎包装成具有产品竞争力的 AVG 画布或跑团体验端。

## 预估周期
3 - 4 周

## 第一周：开局/创角流程前端开发
* **目标：** 从产品入口即抓住用户，使其顺利无缝进入不同的小说维度。
* **具体任务：**
  * [ ] **作品选单页开发**：列出目前系统内已经通过 M1 引擎入库的小说资源库。
  * [ ] **模式分发引导设计**：明确阐述“原著体验”与“自由探索”的差异。
  * [ ] **角色创建器 (Character Creation)**：
    * 列表罗列可以附身的“已有 NPC”。
    * 支持自主输入原创角色的描述（姓名、来历、长板短板、持有物品）。后台要能将这段文本有效化为该 Zep Session 的 `Core Memory`。

## 第二周：主交互场景开发（前端跑团界面）
* **目标：** 摒弃传统的左右互相对话气泡，转向更有叙事氛围的呈现。
* **具体任务：**
  * [ ] **主体布局**：设计成双区布局。左边大面积区域用来展示环境描述和流式对话文本，右栏或顶部保留角色的“基础属性面板”或“图鉴收集”。
  * [ ] **富文本与特效支持**：对 Director 返回的不同标签（对话、心理、旁白）采用不同的字体颜色、斜体或动画效果。
  * [ ] **快捷输入器**：为手机和 PC 提供快捷动作输入面板。

## 第三周：上帝视角改写工具 (Creator Panel)
* **目标：** 迎候“二次创作者”（同人文写手），提供大纲改写支持。
* **具体任务：**
  * [ ] **剧情节点管理器开发**：使用树状结构列出当前时间线。
  * [ ] **段落大纲生成器**：允许创作者选择一个节点，在文本框里抛出“这里男主掉下悬崖获得了上古秘籍”，由系统结合前文与该指令，生成几千字的网文连载段落。

## 第四周：用户反馈收集体系与细节打磨
* **目标：** 完善可用性，防止死局。
* **具体任务：**
  * [ ] **系统提示与重置功能**：提供 `/rewind`（退回上一步）、`/summary`（强制让 AI 当前世界总结）等高级命令用于救场。
  * [ ] **UI/UX 反复调优**：字体排印优化、背景切换逻辑开发以及交互微动效打磨。

## 验收标准
1. 用户可通过 UI 顺滑完成从角色选择、捏人到处在中世纪小木屋的一整套流程。
2. 界面的文本渲染效果符合一款优质跑团游戏的基础素质。
